# 测试执行模块详细设计

## 1. 模块概述

测试执行模块负责测试流程的控制、任务调度、测试前准备和报告生成。

## 2. 功能需求

- 测试执行：执行指定的测试配置或测试用例
- 测试前准备：执行测试前的准备工作，如发送AT指令设置参数
- 测试停止：停止正在执行的测试
- 状态查询：获取测试执行状态
- 报告生成：生成测试报告

## 3. 设计方案

- 采用Click/Typer框架构建CLI界面
- 实现测试管理器，控制测试流程
- 实现执行引擎，调度测试任务
- 实现结果管理器，收集和处理测试结果
- 提供报告生成器，生成HTML和Excel格式的报告

## 5. 数据结构

```python
# 测试状态枚举
class TestStatusEnum:
    PENDING = "pending"
    RUNNING = "running"
    PASSED = "passed"
    FAILED = "failed"
    STOPPED = "stopped"

# 测试会话
class TestSession:
    session_id: str
    status: str  # 取值：TestStatusEnum枚举值
    start_time: datetime
    end_time: datetime
    test_cases: List[TestCase]
    results: List[TestResult]
    config_file: str
    prep_script: str

# 测试用例
class TestCase:
    case_id: str
    name: str
    description: str
    steps: List[TestStep]
    priority: str  # HIGH, MEDIUM, LOW
    timeout: int  # 测试用例超时时间（秒）

# 测试步骤
class TestStep:
    step_id: str
    name: str
    action: str  # 执行的操作，如：SCAN, CONNECT, SEND_DATA
    parameters: dict  # 操作参数
    expected_result: str
    timeout: int  # 步骤超时时间（秒）

# 测试结果
class TestResult:
    case_id: str
    status: str  # 取值：TestStatusEnum枚举值
    start_time: datetime
    end_time: datetime
    actual_result: str
    error_message: str
    log_path: str
    step_results: List[StepResult]

# 步骤结果
class StepResult:
    step_id: str
    status: str  # 取值：TestStatusEnum枚举值
    start_time: datetime
    end_time: datetime
    actual_result: str
    error_message: str

# 测试状态
class TestStatus:
    session_id: str
    status: str  # 取值：TestStatusEnum枚举值
    progress: float  # 0-100
    current_case: str
    current_step: str
    elapsed_time: int  # 已执行时间（秒）

# 测试配置
class TestConfig:
    name: str
    description: str
    devices: List[DeviceConfig]
    test_cases: List[str]
    prep_script: str
    concurrency: int
    timeout: int

# 设备配置
class DeviceConfig:
    device_id: str
    device_type: str  # PHONE, BLE_DEVICE
    connection_info: dict

# 测试会话信息
class TestSessionInfo:
    session_id: str
    status: str  # 取值：TestStatusEnum枚举值
    start_time: datetime
    end_time: datetime
    test_cases: List[str]
    progress: float
```

## 6. 实现细节

- **测试配置解析**：
  - 实现YAML/JSON格式的配置文件解析
  - 支持配置的验证和错误处理
  - 提供配置的默认值和类型转换
  - 支持配置的继承和覆盖
- **测试执行引擎**：
  - 实现测试用例的执行和管理
  - 支持并发执行测试，使用ThreadPoolExecutor
  - 实现任务的调度和优先级管理
  - 支持测试的暂停、恢复和停止
- **模块集成**：
  - 与ADB控制模块集成，获取和控制手机设备
  - 与串口控制模块集成，执行AT指令和控制蓝牙设备
  - 与日志收集模块集成，采集和分析测试日志
  - 与核心引擎集成，获取资源管理和结果管理接口
- **报告生成**：
  - 使用Jinja2模板引擎生成HTML报告
  - 使用openpyxl生成Excel报告
  - 支持自定义报告模板
  - 实现报告的自动分发和通知
- **错误处理**：
  - 实现错误的捕获和处理
  - 支持错误的分类和优先级管理
  - 提供错误的重试和恢复策略
  - 实现错误的日志记录和报告
- **测试前准备**：
  - 支持Python脚本执行测试前准备
  - 支持AT指令序列文件执行设备配置
  - 实现准备脚本的执行和监控

- **测试用例管理**：

## 8. 部署与集成

### 8.1 依赖管理
- **核心依赖**：
  - Click/Typer：用于命令行界面
  - pyyaml：用于配置文件解析
  - loguru：用于日志管理
  - jinja2：用于HTML报告生成
  - openpyxl：用于Excel报告生成
  - pandas：用于数据处理和分析
  - matplotlib：用于数据可视化
- **可选依赖**：
  - pytest：用于单元测试
  - coverage：用于测试覆盖率分析
  - requests：用于网络请求

### 8.2 安装与配置
- **安装方式**：
  - 通过pip安装：`pip install bt-test`
  - 从源码安装：`pip install -e .`
- **配置文件**：
  - 支持JSON、YAML格式的配置文件
  - 默认配置文件路径：`~/.config/bt-test/config.yaml`
  - 支持通过环境变量覆盖配置

### 8.3 集成接口
- **CLI接口**：提供命令行接口，支持各种测试操作
- **Python API**：提供Python API，便于其他Python应用集成
- **Web API**：可选提供Web API，支持远程测试执行
- **与其他模块的集成**：
  - 与ADB控制模块集成，获取和控制手机设备
  - 与串口控制模块集成，执行AT指令和控制蓝牙设备
  - 与日志收集模块集成，采集和分析测试日志
  - 与GUI模块集成，提供状态更新和命令执行接口

### 8.4 部署场景
- **本地部署**：直接在本地机器上安装和运行
- **容器化部署**：支持Docker容器化部署
- **远程部署**：支持在远程服务器上部署，通过网络执行测试
